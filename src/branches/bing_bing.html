Abstract

This project outlines the design and development of a relational database system for a gaming platform named "Bing Bing." The system encompasses entities such as Player, Hero, Skin, Game, and more, establishing relationships through well-defined ERD structures. The ERD was systematically converted into a relational schema, adhering to normalization principles up to 3NF.

Key entities include Player, Hero, Skin, Game_mode, Team, and Personal_statistics, interconnected with relationships like Player-Hero (M:M), Player-Skin (1:M), Team-Game_mode (M:1), and others. Logical, null, unique, primary key, and default constraints were meticulously identified, ensuring data integrity and consistency.

The SQL-DDL section presents the necessary commands for creating databases and tables. Tables are designed to support foreign key relationships, ensuring referential integrity through CASCADE actions. The SQL-DML section illustrates data insertion syntax for various tables.

The document concludes with a workload matrix, showcasing the collaborative effort in completing different components of the assignment. The proposed database model provides a robust foundation for managing player profiles, heroes, game data, and related statistics in the gaming platform.
